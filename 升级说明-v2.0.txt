每日打卡 v2.1 升级说明
======================

版本名称：2.1
更新日期：2026-05-20

一、升级目标
------------
v2.0 将原来的单一打卡应用升级为“单一用户 + 多项目打卡”应用。
用户可以在同一个应用里管理不同打卡场景，例如上班、健身、喝水、吃药、学习等。
v2.1 在此基础上继续补充目标管理、休息日设置、数据导入导出和日历状态展示。

二、核心功能变化
----------------
1. 多项目打卡
   - 默认提供两个项目：
     - 上班：签到/签退模式
     - 健身：单次打卡模式
   - 每个项目的打卡记录独立保存。
   - 首页可以直接切换当前项目。

2. 首页新增添加项目入口
   - 首页打卡卡片中新增“+”添加项目按钮。
   - 用户不需要进入设置页，也能直接创建新项目。
   - 创建项目已改为自定义弹窗，不再使用浏览器 prompt。
   - 创建后会自动切换到新项目，方便立即测试或打卡。

3. 两种打卡模式
   - 签到/签退模式：
     - 记录开始时间和结束时间。
     - 自动计算时长。
     - 适合上班、学习、训练等需要统计耗时的场景。
   - 单次打卡模式：
     - 只记录完成时间。
     - 不计算时长。
     - 适合喝水、吃药、完成习惯等场景。

4. 项目管理
   - 设置页保留项目管理能力。
   - 支持添加、删除、重命名项目。
   - 支持设置每周目标、每月目标和每日目标时长。
   - 支持设置每周固定休息日和自定义休息日期。
   - 项目模式仅在创建时选择，创建后不可修改，避免历史数据统计口径混乱。

5. 数据安全
   - 设置页支持导出 JSON 备份。
   - 设置页支持导出 CSV 记录。
   - 设置页支持导入 JSON，并会在导入前确认覆盖当前本地数据。

三、统计升级
------------
1. 首页统计
   - 首页统计会跟随当前选中的项目变化。
   - 不同项目之间的数据互不影响。

2. 统计页
   - 新增“项目统计”模块。
   - 按项目展示：
     - 打卡天数
     - 完成次数
     - 累计时长（仅签到/签退模式显示）
   - 原有周统计、趋势图、月度统计、平均数据继续展示当前项目的数据。

3. 日历视图
   - 日历默认展示。
   - 支持显示已完成、进行中、过去未打卡、项目休息日和内置节假日。
   - 当前内置节假日只包含少量 2026 年示例日期，后续可继续补全。

四、兼容性处理
--------------
1. 旧数据迁移
   - 旧版本保存在全局 checkin_records 中的记录会自动归入默认“上班”项目。
   - 升级后不会丢失原有打卡记录。

2. 离线缓存
   - Service Worker 缓存版本已升级到 checkin-cache-v12。
   - 如果线上页面仍显示旧内容，建议强制刷新或关闭页面后重新打开。

五、已修复问题
--------------
1. 修复 Toast 的 innerHTML 注入风险。
2. 修复本周统计起始日不一致问题，统一从周一开始。
3. 支持跨天签退和夜班场景。
4. 修复近 30 天趋势空状态显示错位问题。
5. 将项目创建入口从设置页前移到首页，提高可发现性。
6. 首页添加项目从 prompt 改为自定义弹窗。
7. 设置页清空记录移动到“危险操作”区域，并保留二次确认。

六、测试建议
------------
1. 首次打开应用后，确认默认项目包含“上班”和“健身”。
2. 在“上班”项目测试签到和签退，确认能统计时长。
3. 在“健身”项目测试单次打卡，确认只记录完成时间。
4. 使用首页“添加打卡项目”创建新项目，分别测试两种模式。
5. 在设置页导出 JSON、导出 CSV，确认文件可以生成。
6. 设置项目目标，确认首页项目标签出现目标摘要。
7. 设置项目休息日，确认日历显示“休”。
8. 进入统计页，确认各项目统计相互独立。
